ShellPay API

Note When sending POST requests, please sign the raw request data using SHA256-HMAC with your private key and place this signature in the 'HMAC-SIGNATURE' header. This is how we authenticate your requests. If you omit or send an invalid signature, you will receive an error

API Root: https://shellpay/api

POST /api/keys

Creates public and private keys. Keys expire after the expiry date

Parameters Description Type
address Your TurtleCoin address, standard or integrated address string
Example Payload 

{
  "address": "TRTLuySpDqd2fcvq5vx7Jiayw6yao7JHXFPuia5V83cVREtQSKyvWpxX9vamnUcG35BkQy6VfwUy5CsV9YNomioPGGyVhK3YXLq"
}
Response 

{
  "status": "ok" or an error message ,
  "result": {
    "expiry": # key expiration date (unix timestamp)
    "private_key": 128 character string, # your private key
    "public_key": 64 character string # your public key
  }
}
POST /api/invoice

Creates a payment invoice

Parameters Description Type
ipn_uri This is where ShellPay will send the payment notification for this invoice string
currency_code The original currency of the transaction. See supported currency codes string
currency_amount The transaction cost in the original curency float64
order_id Merchant Specified id for this transaction string
custom Merchant Specified custom data string
public_key The public key of a merchant string
Example Payload 

{
    'ipn_uri': 'https://starbucks.com/payments/',
    'currency_amount': 10,  
    'currency_code': 'usd',
    'order_id': '123456',   
    'custom': "{'item_name': 'one tall coffee'}",
    'public_key': "799ca1122df0f885a9f0a256129856e1ba3b001230332e6ea2a5912e8a2caf1e"
}
Response 

{
    "status": "ok" or error message,
    "result":
    {
        "status": invoice status - unpaid/processing/expired/paid,
        "timestamp": the current unix timestamp,
        "payment_id": 64 character string, you can use this value later to call GET /api/invoice/:id,
    }
}
GET /api/invoice/:invoice_id

Retrieves invoice information.

Response 

{
    "status": "ok" or error message,
    "result":
    {
        "status": invoice status - unpaid/processing/expired/paid,
        "amount_received": the atomic amount of TRTL received,
        "atomic_amount": the atomic amount of TRTL due,
        "currency_amount": amount due in the original currency,
        "currency_code": the code of the original currency,
        "order_id": id of the order,
        "expiration": timestamp of the invoice expiration,
        "payment_address": the address where funds should be sent,
        "fee": dev fee amount in TRTL atomic units,
    }
}

Payment Notifications

These notitifications are sent to the ipn_uri specified during invoice creation.


Response 

{
    "type": invoice status types = invoice_created, paid, expired:,
    "currency_amount": amount due in the original currency,
    "currency_code": the currency code,
    "custom": Merchant specified custom data.
    "merchant_address": the merchant's address,
    "order_id": the order id,
    "public_key": merchant's public key,
}
